然后谈另外一个主要问题
我自己写的project 上传到git上 出现不完整
比如说 你有很多个project 比如叫 CAS 或update excel 什么的，然后 git它会默认把你一些东西清除掉 
 就是 本来
我偷懒 上传到同一个 git 仓库
我画红线的是 git把我删除了的文件夹
我在git上面 没有看到这三个文件夹
意思就是说 一个git的仓库只默认一个project 
其他的project 它不会给你上传的
如果我重新 做一个新的仓库 该有的东西他都有（特殊情况未考虑）
就是 我重新建一个仓库 单独上传那个update excel project 刚刚我画的那三个文件夹就会有 
我就只能认为是 一个仓库只能放一个project 
如果一个仓库放了两个以上 它会默认去掉其他的project的bin 或一些 import project的文件夹
那个 gitignore 应该不是干这个事的人 
不过听说没有上传的那几个形如bin的文件是eclipse生成的文件，并不是很重要的组成部分，其他部分仍然可以编译，问题跟进中。。。
